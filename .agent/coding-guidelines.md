# Coding Guidelines

- Escreva C# limpo e direto.
- Use nomes explicitos.
- Mantenha classes pequenas.
- Evite logica pesada em `MonoBehaviour` quando possivel.
- Separe configuracao, UI, navegacao e player.
- Documente decisoes tecnicas relevantes.
- Nao quebre a estrutura de pastas existente.
- Nao adicione backend sem necessidade.
- Nao implemente multiplayer nesta PoC.
- Nao implemente quiz antes da fase planejada.
- Nao commite videos grandes reais no Git sem decisao explicita.
- Mantenha compatibilidade com Meta Quest 3 e Unity/OpenXR.
- Preserve fallback de desenvolvimento no Editor.
