# Riscos

| Risco | Mitigação |
| --- | --- |
| Cliente pedir jogo completo, mas escopo real ser player 360. | Manter a PoC focada em menu + player e documentar limites antes de estimar MVP. |
| Vídeos grandes demais. | Testar vídeo curto real, medir tamanho, bitrate, carregamento e performance no Quest 3. |
| Falta de Quest 3 para teste. | Não prometer compatibilidade final antes do headset; separar validação sem headset e com headset. |
| Internet instável em exposição. | Priorizar app Unity com suporte offline ou definir cache/download controlado. |
| Crianças se confundirem com interface. | Usar botões grandes, poucos elementos e teste com monitor/professor. |
| Desconforto em VR. | Evitar locomotion artificial e validar duração/orientação no headset real. |
| Direitos autorais dos vídeos. | Usar apenas conteúdo autorizado e registrar origem/licença. |
| Diferença entre WebXR e app nativo. | Tratar web demo como comparação, não como substituto sem teste. |
| Expectativa de publicação em loja sem necessidade. | Confirmar se sideload/APK basta para exposição. |
| Operação manual difícil em evento. | Validar fluxo de professor, reinício de sessão e troca rápida entre vídeos. |
