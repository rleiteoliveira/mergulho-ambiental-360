# Opções Técnicas

## A) App nativo Unity para Quest 3

### Vantagens

- Melhor controle da experiência.
- Possibilidade de uso offline.
- Melhor para exposição.
- Interface fechada.
- Melhor integração XR.
- Mais adequado para crianças.

### Desvantagens

- Exige build.
- Exige teste no headset.
- Setup mais complexo.
- Atualização de vídeos pode exigir processo definido.

## B) Site/WebXR

### Vantagens

- Rápido.
- Acessível por link.
- Fácil atualizar vídeos.
- Pode ser usado também em notebook/celular.
- Bom para demo.

### Desvantagens

- Depende de navegador.
- Pode depender de internet.
- Menor controle.
- Menos confiável para exposição.
- Cache/offline de vídeos grandes pode ser problema.

## C) Híbrido

- App Unity para exposição.
- Site para divulgação, catálogo e preview.
- Site pode ajudar no alinhamento com cliente.
- Web demo pode validar linguagem visual e organização antes do build nativo.

## Recomendação inicial

Para entrega principal em exposição infantil, começar com app Unity. Manter `web-demo/` apenas como prova comparativa, catálogo visual ou landing/demo.

A decisão final depende de teste no Quest 3 real e de respostas do cliente sobre internet, operação, quantidade de headsets, atualização de conteúdo e expectativa de entrega.
